## Code of Conduct

This project is intended to be a safe, welcoming space for collaboration. Please be respectful and constructive in all interactions.

## How Can I Contribute?

### üêõ Reporting Bugs

If you find a bug, please open an issue with:
- A clear, descriptive title
- Steps to reproduce the problem
- Expected vs actual behavior
- Screenshots if applicable
- Browser and device information

### üí° Suggesting Features

Feature suggestions are welcome! Please include:
- A clear description of the feature
- Why it would be useful for the target audience (kids 8-12)
- Any mockups or examples if applicable

### üé® Improving Content

Help make the educational content better:
- Simplify explanations
- Fix typos or grammatical errors
- Suggest better analogies for complex concepts
- Add more kid-friendly examples

### üåç Translations

Help make this accessible to more children worldwide:
- Translate content to other languages
- Ensure translations are age-appropriate
- Maintain the fun, engaging tone

### üíª Code Contributions

#### What We're Looking For
- Bug fixes
- Performance improvements
- Accessibility enhancements
- New interactive elements
- Mobile responsiveness improvements

#### What to Avoid
- Adding heavy frameworks or dependencies
- Changes that make content less kid-friendly
- Features that require backend servers

## Style Guidelines

### HTML
- Use semantic HTML5 elements
- Keep markup clean and readable
- Include alt text for all images

### CSS
- Use CSS custom properties (variables) for colors
- Follow the existing naming conventions
- Mobile-first responsive design

### JavaScript
- Vanilla JS only (no frameworks)
- Clear, commented code
- Keep it simple and maintainable

### Content
- Write at a 4th-5th grade reading level
- Use short sentences
- Include fun analogies kids can relate to
- Avoid jargon without explanation

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create** a new branch for your changes
4. **Make** your changes
5. **Test** in multiple browsers
6. **Commit** with clear messages
7. **Push** to your fork
8. **Open** a Pull Request

### Commit Message Format

```
type: short description

Longer description if needed.
```

Types:
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation
- `style`: Formatting, CSS
- `refactor`: Code restructuring
- `test`: Adding tests
- `chore`: Maintenance

### Example
```
feat: add Spanish translation for page 1

Translated all text content on the cover page
to Spanish. Maintained the playful tone of the
original content.
```
